¿Qué es un Sistema de Diseño?

Un Sistema de Diseño es el conjunto de estándares, componentes y guías que permiten a un equipo construir productos con consistencia visual y funcional a escala.

UI Kit vs Sistema de Diseño

UI Kit: la colección visual de componentes en Figma.

Sistema de Diseño: el UI Kit + la documentación + las reglas de uso + los principios de marca.

Para su proyecto: construirán el UI Kit. En una empresa, el equipo mantiene el sistema completo.

Ejemplos reales

  • Material Design 3 (Google) — Android + Web
  • Apple HIG — iOS + macOS
  • Fluent Design (Microsoft) — Windows
  • Atlassian Design System — Jira, Confluence

💡 ¿Por qué importa? Sin un UI Kit, cada pantalla que diseñen puede verse diferente. Con un UI Kit, cambiar el color primario actualiza todos los componentes en segundos.

Design Atómico — La estructura de un UI Kit

Brad Frost (2013) propone organizar los componentes de menor a mayor complejidad, igual que la química: átomos → moléculas → organismos → plantillas → páginas.

Color con propósito — Design Tokens

Un design token es un nombre semántico para un valor de diseño. En lugar de usar #bd93f9 directamente, usas --color-primary. Si cambias el token, cambia todo.

Regla de los 3 colores

  • Primario — acciones principales, CTA, links
  • Secundario — apoyo, variantes, acentos
  • Neutro — fondos, textos, bordes

+ colores semánticos: success, error, warning, info

WCAG — Contraste mínimo

  • Texto normal: ratio ≥ 4.5:1 (AA)
  • Texto grande (18px+): ratio ≥ 3:1 (AA)
  • Herramienta: Figma Plugin "Contrast"

💡 En Figma: Crear colores como Styles o Variables. Al cambiar el valor del estilo, se actualiza en todos los componentes que lo usan.

Tipografía con jerarquía

La tipografía no es decoración — es arquitectura de la información. Una jerarquía clara guía al usuario por la pantalla sin que tenga que decidir qué leer primero.

Reglas para móvil

  • Cuerpo mínimo 16px (recordar S16)
  • Línea de texto: máximo 60–70 caracteres
  • Interlineado: 1.4–1.6× el tamaño de fuente
  • Máximo 2 fuentes en el mismo proyecto

En Figma — Text Styles

  • Crear un estilo por cada nivel: H1, H2, Body, Caption, Label
  • Definir: familia, peso, tamaño, interlineado, tracking
  • Aplicar desde el panel de estilos, no manualmente

💡 Fuentes gratuitas recomendadas: Inter (neutralidad), Sora (modernidad), Plus Jakarta Sans (legibilidad móvil).

Componentes y sus estados

Un componente en Figma es una pieza reutilizable. Cada componente debe tener definidos sus estados — la interfaz no siempre está en el estado "ideal".

💡 En Figma: Usar Properties para crear variantes de un mismo componente. Un componente "Botón" puede tener variante por tipo (primario/secundario) y estado (default/hover/disabled).

Guías de diseño — Sus referencias obligatorias

Las guías de diseño son los estándares definidos por las plataformas. Para su proyecto, revisen ambas — están diseñando para Web y Móvil.

🤖 Material Design 3 (Google)

m3.material.io

  • Sistema de colores con Dynamic Color
  • Componentes para Android + Web
  • Tipografía: sistema de roles (Display, Headline, Body, Label)
  • Iconos: Material Symbols (variable)
  • Elevación con sombras y tonos de superficie

Ideal si su proyecto es una app Android o web con estilo Google.

🍎 Apple Human Interface Guidelines

developer.apple.com/design/hig

  • Diseño para iOS/iPadOS/macOS/watchOS
  • SF Symbols — más de 6,000 íconos integrados
  • Blur y transparencias (vibrancy)
  • Gestos nativos de iOS (swipe back, pull to refresh)
  • Énfasis en jerarquía y claridad

Ideal si su proyecto es una app iOS.

🎨 UI Patterns

ui-patterns.com

  • Colección de patrones de interacción probados
  • Problema → Solución → Ejemplos reales
  • Búsqueda por tipo de problema (navegación, formularios, etc.)

Útil cuando no saben cómo resolver un problema de interacción específico.

🔑 Para su proyecto — ¿qué aplicar?

  • No copien estilos de Material o Apple — tomen principios
  • Decidan: ¿su app tiene personalidad de marca propia o sigue un sistema?
  • Justifiquen sus decisiones con referencia a las guías en la entrega
  • Prioridad: consistencia interna > seguir una guía al pie de la letra

Espaciado, grids y radios

Sistema de espaciado base-4

Usar múltiplos de 4px: 4, 8, 12, 16, 24, 32, 48, 64. Esto garantiza consistencia y facilita las implementaciones en código.

¿Por qué 4? Es el GCD de la mayoría de resoluciones de pantalla y las fuentes.

Grids

  • Móvil: 4 columnas, margen 16–20px, gutter 8–12px
  • Tablet: 8 columnas, margen 24px, gutter 16px
  • Escritorio: 12 columnas, margen 40–80px, gutter 24px

Border Radius

  • none: 0px — elementos de sistema, separadores
  • sm: 4px — inputs, tags pequeños
  • md: 8px — cards, botones
  • lg: 12–16px — modales, sheets
  • full: 9999px — chips, avatares
